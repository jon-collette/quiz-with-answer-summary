How easy-to-use is your interface?

I've taken to calling this the 10x UI test — it's most useful to reevaluate how effective (or frustrating) your UI is by doing the same action ten times in a row. I've provided scenarios with good and bad examples to test this theory on and thoughts on each.

Scenario one

Date pickers vs simple inputs

As seen in the examples, datepickers excel at showing the user relative dates and secondary information. Great use scenarios are:

  • Google Flights: showing price differences between dates (flight cost 9/21 vs 9/23)
  • Workday PTO scheduling: selecting relative through dates (next Wednesday through Friday)
  • Restuarant reservations: selecting day of week rather than actual dates (this Thursday, not 09/23/2021)

Datepickers have their uses but are incredibly annoying to use when implemented wrong.

Scenario two

Hover- vs click-opening menus

Hover-opening menus continue to be used across the web and prove to be frustrating in 2021. Here's a few key issues:

  • Divergence from touchscreen experience: Menu behavior acts differently between a touchscreen and mouse-driven device, why not simplify and merge the experiences?
  • Coding required and accessibility issues: most CSS frameworks default to click-based methods in 2021 and come with preset ARIA functions — changing to a hover-based menu is haphazard at best.
  • Extra pages potentially required: Are dropdown parent items actual pages? Do their content only consist of links and descriptions of the child pages? What user needs to see a second list?
  • Unexpected screen behavior: Menus opening unexpectedly are frustrating, especially when they cover content.
